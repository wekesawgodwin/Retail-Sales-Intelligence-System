# Project Plan: Retail Sales Intelligence System

## 1. Project Overview
This project analyzes retail sales data to identify trends in sales, profit, customer segments, product performance, and regional performance. The work will be completed using Python notebooks, SQLite, SQL queries, and data visualization tools.

## 2. Project Goals
- Clean and prepare the dataset for analysis.
- Engineer useful features for deeper insights.
- Store the processed data in a SQLite database.
- Perform exploratory analysis using SQL and Python.
- Create clear visualizations that support findings.
- Summarize business insights and recommendations.
- Demonstrate teamwork and GitHub collaboration.

## 3. Tools and Technologies
- Python
- Jupyter Notebook
- pandas
- NumPy
- matplotlib
- seaborn
- SQLite
- Git and GitHub
- Anaconda

## 4. Dataset
The team will use a public retail sales dataset saved as `retail_sales_raw.csv` in `data/raw/`. The dataset should contain key fields such as order date, sales, profit, category, sub-category, region, customer segment, and product name.

## 5. Main Questions
- Which categories generate the highest sales and profit?
- Which regions perform best?
- Which products are most profitable?
- How do sales change over time?
- What is the effect of discounts on profit?
- Which customer segments contribute most to revenue?

## 6. Project Phases
### Phase 1: Setup
- Create repository and folder structure.
- Add team members and assign roles.
- Upload raw dataset.

### Phase 2: Data Cleaning
- Inspect missing values and duplicates.
- Standardize column names.
- Fix data types.
- Save the cleaned dataset.

### Phase 3: Feature Engineering
- Create year, month, weekday, quarter, and profit margin columns.
- Save the processed dataset.

### Phase 4: SQL Analysis
- Load processed data into SQLite.
- Write SQL queries for business questions.
- Save queries in `queries.sql`.

### Phase 5: Visualization and EDA
- Create charts for sales, profit, region, category, and trends.
- Save charts in `outputs/charts/`.

### Phase 6: Insights and Reporting
- Summarize findings.
- Write recommendations.
- Finalize README and presentation notes.

## 7. Expected Deliverables
- Cleaned dataset
- Processed dataset
- SQLite database
- SQL query file
- Analysis notebook
- Visualizations
- Insights report
- README file

## 8. Timeline
The project will be completed in one week, with one hour of work per day per member. Each member must complete their assigned task by the end of the day so the next phase can begin smoothly.

## 9. Collaboration Rules
- Work on assigned notebooks only.
- Use Git branches for changes.
- Commit frequently with clear messages.
- Review work before merging.
- Keep file names and folder paths consistent.
